# Retitle the writer's CLI usage mandate to an owning-verbs mandate: invariant first, then the plan CLI verbs named once as this persona's execution path with the enumeration compressed to the verb-group list, leaving the tools allowlist byte-identical

## Scope

- `src/vaultspec_core/builtins/agents/vaultspec-writer.md`

## Description

- Retitle the writer's CLI usage mandate to an owning-verbs mandate.
- Lead with the invariant: every structural manipulation routes through the owning plan verbs, never hand-edits, with the canonical-identifier and gap-no-reuse guarantee.
- Name the vault plan CLI once as this persona's execution path.
- Compress the per-verb enumeration into the verb-group list (step, phase, wave, epic intent, tier).
- Leave the tools allowlist byte-identical.

## Outcome

- The writer mandate now separates the permanent owning-verb invariant from the CLI transport that carries it, keeping every exact verb and the byte-identical allowlist.

## Notes

- Capability-first, not MCP-first: the persona is a dispatched subagent whose allowlist holds no MCP tool, so the CLI stays the named mechanism.
